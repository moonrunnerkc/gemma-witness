# Release checklist

CI proves the wiring on three platforms on every push via the hermetic e2e
against `witness-test-sidecar`. It does not prove the capture binary still
drives a real Gemma 4 sidecar end-to-end; the GitHub macOS runner cannot
host the model. The maintainer closes that gap manually before every tagged
release.

## Trust anchors

The distribution chain depends on two out-of-band facts that the binary
cannot self-attest. Both are now satisfied on `main`; this section
documents them and the conditions under which they must be re-established.

### S-1: Signed release artifacts

The signed release pipeline lives in `.github/workflows/release.yml`. It
runs only on `push: tags: ['v*']`, publishes SHA-256 of each artifact in
`SHASUMS256.txt`, signs that file via Sigstore cosign (keyless OIDC), and
attaches SLSA v1 build provenance.

Sigstore keyless cosign produces no long-lived public key; the trust
anchor is the OIDC certificate identity that GitHub Actions presents to
Fulcio at signing time. Verifiers MUST pin both fields below, otherwise
a swap of the entire release page is undetectable.

```
Cosign signing identity (keyless / Sigstore OIDC):
  certificate-identity-regexp: ^https://github\.com/moonrunnerkc/gemma-witness/\.github/workflows/release\.yml@refs/tags/v.+$
  certificate-oidc-issuer:     https://token.actions.githubusercontent.com
```

Update this block if the repository moves owners, the workflow file is
renamed, or the tag pattern changes. The release notes generated by
`release.yml` already echo these fields in the verification incantation;
they MUST stay in sync with this file.

### S-5: MLX fingerprint provenance

`inference/fingerprints/mlx-community__gemma-4-e4b-it-4bit__cc3b666c.json`
ships with `verified_by: "huggingface-lfs+local-recompute"`, meaning the
SHA-256 was cross-checked against the Hugging Face LFS oid for the pinned
revision. Re-run the seeder before any release that bumps the model
revision, and on any host where the local model cache may have drifted:

```bash
cargo run -p seed-fingerprints -- \
  --model-id mlx-community/gemma-4-e4b-it-4bit \
  --revision cc3b666c01c20395e0dcebd53854504c7d9821f9
```

The seeder fetches the HF LFS oid, recomputes SHA-256 against the local
`model.safetensors`, and refuses to overwrite the entry on mismatch. Do
not pass `--force`; mismatches are an investigation, not a workaround.
Commit the regenerated JSON and the regenerated
`apps/verifier/known-fingerprints.json` so CI picks them up. If the
seeder cannot run (HF outage, no cached weights), do not tag the release
until it can; the registry's provenance label is part of the trust chain.

## Pre-tag steps

Run on Apple Silicon with the pinned MLX model cached:

```bash
# 1. Bring up the live sidecar.
inference/mlx-sidecar/start.sh

# 2. Live e2e against the real model. Must pass, not skip.
cargo test -p witness-core --test day-4-e2e -- --nocapture

# 3. Full workspace tests once more for the record.
cargo test --locked --workspace -- --test-threads=1

# 4. Clippy + fmt gate. Must be clean.
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo fmt --all -- --check

# 5. Supply-chain audit.
cargo audit --locked
cargo deny --locked check advisories bans sources
( cd apps/verifier && pnpm audit --audit-level moderate )
( cd apps/capture && pnpm audit --audit-level moderate )

# 6. Verifier build. Output should be a single HTML file.
cd apps/verifier && pnpm install && pnpm build && cd -

# 7. Spot-check the verifier with the bundle the live e2e produced.
#    Drag it into apps/verifier/dist/verify.html. Expect every check green
#    AND a "Signed by a known witness" row in green; otherwise add the
#    release signer's fingerprint to apps/verifier/trusted-signers.json
#    first.
```

If any step fails or `day-4-e2e` hits its skip path instead of running, do
not tag. The skip path exists for CI on hosts that cannot run the model; on
a release host it is a bug.

## Tagging

```bash
git tag -s v0.X.Y -m "v0.X.Y"
git push origin v0.X.Y
```

## After tagging

- Confirm the GitHub Actions run for the tag (both `CI` and `Release`) is green.
- Confirm `SHASUMS256.txt`, `SHASUMS256.txt.sig`, and `SHASUMS256.txt.crt`
  attach to the release.
- Update `apps/verifier/known-fingerprints.json` if any model fingerprint
  was added or rotated in this release; the verifier ships with the list
  embedded.
- Update `apps/verifier/trusted-signers.json` if any new release signer
  needs to be registered.
- File a release-notes entry naming the manifest version and any changes to
  the bundle layout. The manifest version did not change in this release
  unless explicitly noted.

## When the live e2e cannot run

A self-hosted Apple Silicon runner would make this automated. The
maintenance burden of a Mac mini sitting in a closet is real, and the
hermetic e2e already catches the schema-drift class of bugs on every push,
so the live e2e stays as a release gate, not a per-push gate. If a
self-hosted runner lands later, replace this checklist with a CI job; no
migration cost beyond deleting this file.
